JS Mode Basics

To use JavaScript in any object, start your script with the // @javascript header. In this mode, your script is executed as an async function with full access to the app instance and the handler name (e.g. "mouseUp" or "idle").

Header & Event Handling
// @javascript
if (handler !== "mouseUp") return;

alert("This script only runs on mouseUp!");

Navigation

Use the app.goToCard() method to navigate. It accepts card IDs, names (lowercase), or keywords like "next", "prev", "first", and "last".

Navigating Cards
// @javascript
await app.goToCard("next");
await app.goToCard("home");
await app.goToCard("back");

State & Data

You can read and modify the entire stack state directly via app.state. This is the equivalent of put and get in HyperCode.

Reading State
// @javascript
const cardCount = app.state.cards.length;
const currentCardName = app.state.currentCard.name;
const itValue = app.state.it; // Global "it" container

// Setting the "it" value manually
app.state.it = "Saved from JS";
Idle (Update Loop)

The idle handler fires roughly every 300ms. Use it for constant background logic or persistent animations.

// @javascript
if (handler !== "idle") return;
// Your periodic logic here...

UI & Dialogs

Standard browser dialogs work, or you can use the built-in app.dialogs for complex interactions.

Dialog Examples
// @javascript
const name = prompt("What is your name?");
if (name) {
    alert("Hello, " + name);
}

// Triggering built-in stack info
app.dialogs.openStackInfo();

Network & URL (Fetch)

One of the most powerful features of JS mode is the ability to fetch external data and display it in your stack.

Retrieve URL into Field
// @javascript
if (handler !== "mouseUp") return;

try {
    // Fetch some plain text from a URL
    const response = await fetch("https://baconipsum.com/api/?type=meat-and-filler&sentences=1&format=text");
    const text = await response.text();

    // Find field named "News" and set its content
    const field = app.findObjectByName("News", "field");
    if (field) {
        field.content = text;
        // CRITICAL: Must trigger UI update
        app.ui.renderCard();
        app.scheduleAutoSave();
    }
} catch (e) {
    alert("Fetch failed: " + e.message);
}

Object Manipulation

In JS mode, you must find the object model first, modify its properties, and then tell the UI to re-render.

Setting Object Properties
// @javascript
// Note: findObjectByName is available on the engine/app
const btn = app.findObjectByName("Submit", "button");
if (btn) {
    btn.visible = false;
    btn.title = "Processing...";
    
    app.ui.renderCard();
}

Visuals & Animation

You can trigger the same transitions and animations used by the main language.

Transitions & Effects
// @javascript
// Set transition for next card move
app.state.nextTransition = "dissolve";
await app.goToCard("next");

// Animate a specific element
const logo = app.findObjectByName("Logo");
if (logo) {
    // Access the DOM element to apply CSS animations
    const el = document.querySelector(`[data-id="${logo.id}"]`);
    if (el) {
        el.classList.add("hc-anim-bounce");
        setTimeout(() => el.classList.remove("hc-anim-bounce"), 1000);
    }
}
