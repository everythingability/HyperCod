Introduction

HyperCode is an English-like scripting language inspired by HyperTalk. It is designed to be highly readable, allowing you to create complex interactions with simple "natural" commands. Every script in HyperCod consists of handlers that respond to messages like clicking a button or opening a card.

Syntax Basics

Scripts are organized into handlers. A handler starts with on followed by the message name, and ends with end followed by the same name.

Basic Handler Structure
on mouseUp
  beep
  answer "You clicked me!"
end mouseUp

You can add comments to your code using two dashes --. Anything after the dashes on that line is ignored by the engine.

-- This is a comment
beep -- This beeps once

Navigation

The go command is used to move between cards in your stack.

Command Description
go next Move to the next card.
go previous Move to the previous card.
go first Move to the first card in the stack.
go last Move to the last card in the stack.
go to card "Home" Go to a card named "Home".
go card 5 Go to the 5th card in the stack.
go back Return to the card you were previously on.

Handling Data

The put and get commands are the core of data movement. put can store values into variables or fields.

Examples of Data Handling
-- Putting text into a field
put "Welcome to the stack" into card field "Welcome"

-- Concatenating strings with &
put "Today is " & the date into card field 1

-- Using variables
put 50 into myWealth
put myWealth * 2 into myWealth
answer "Total wealth: " & myWealth

-- Appending and Prepending
put "!" after card field "Notes"
put "Breaking: " before card field "News"

-- The special "it" variable
get the number of cards
answer "We have " & it & " cards!"

Global Functions & Properties

  • the date: Current calendar date.
  • the time: Current system time.
  • the number of cards: Total count of cards in stack.
  • the name of card: Name of the current card.
  • the id of card: Unique ID of the current card.
  • the number of card: Index of the current card.
  • random(n): Returns a random number between 1 and n.
  • the width: The width of the card.
  • the height: The height of the card.
  • the clickText: The text of the field that was just clicked.

User Interaction

Interact with the user via dialogs and sounds.

Interaction Commands
-- Simple alert
answer "Process complete!"

-- User input
ask "Please enter your name:"
put it into card field "Username"

-- Sound
beep

-- Debugging (Developer Console)
log "New position is: " & myLoc

You can also open external links using open url:

open url "https://www.google.com"

Logic & Control Flow

HyperCod supports standard conditional logic and loops.

Conditional (If/Then)

on mouseUp
  ask "What is 2 + 2?"
  if it is 4 then
    answer "Correct!"
  else
    answer "Try again."
  end if
end mouseUp

Loops (Repeat)

-- Repeat a fixed number of times
repeat 5 times
  beep
  wait 1 second
end repeat

-- Loop with a counter
repeat with i = 1 to 10
  put i into card field "Counter"
  wait 10 -- ms
end repeat

-- Loop while a condition is true
put 1 into x
repeat while x < 100
  put x into card field "Stats"
  put x + 7 into x
end repeat

Objects & Properties

You can change the appearance and behavior of buttons, fields, and cards dynamically using set, show, and hide.

Commands

  • show button "Play"
  • hide card field 2
  • set visible of button 1 to false
  • set title of button "Start" to "Finish"
  • set content of field "Display" to "New Text"
  • set backgroundColor of card to "blue"

Supported Properties

Property Applies To Usage
name All Objects the name of button 1
title Buttons Visible text on a button.
visible Buttons, Fields true or false.
enabled Buttons Whether a button can be clicked.
content Fields The text inside the field.
backgroundColor Cards CSS color string (e.g. "red", "#ff0000").
left, top All Objects X and Y coordinates (e.g. set the left of me to 10).
width, height All Objects Dimensions in pixels.
loc All Objects The center point (location) as "x,y".

Visual Effects & Animation

Add polish to your stack with transitions and object animations.

Card Transitions

Set a transition before navigating to another card.

on mouseUp
  visual effect "dissolve"
  go next
end mouseUp

Available effects: dissolve, wipe-left, wipe-right, slide-up, slide-down, zoom-in, zoom-out.

Object Animation

Animate a specific object on the screen.

on mouseUp
  animate button "Star" with "bounce"
end mouseUp

Available animations: bounce, pulse, shake, swing, fade-in, scale-up.

Idle & Background Motion

The idle handler runs roughly every 300ms, making it perfect for autonomous movement.

Walking Image Example
on idle
  -- 1. Get current position
  put the left of me into myLeft
  put the top of me into myTop
  
  -- 2. Add random movement (-20 to +80 pixels)
  put myLeft + (random(100) - 20) into myLeft
  put myTop + (random(100) - 20) into myTop
  
  -- 3. Wrap around screen edges
  if myLeft > the width then put -50 into myLeft
  if myLeft < -50 then put the width into myLeft
  
  if myTop > the height then put -50 into myTop
  if myTop < -50 then put the height into myTop
  
  -- 4. Apply new position and log it
  set the left of me to myLeft
  set the top of me to myTop
  log "Co-ords: " & myLeft & "," & myTop
end idle

JavaScript Integration

For advanced users, you can use raw JavaScript by starting your script with // @javascript. In this mode, you have direct access to the app engine.

JavaScript Example
// @javascript
const cards = app.state.cards;
app.ui._pickImageFile((dataUrl) => {
  app.state.currentCard.backgroundImage = dataUrl;
  app.ui.renderCard();
});
